The aggregator integration has its own contract (Spark-hosted `/api/upgaming/*` endpoints plus the aggregator's wallet surface). It is not covered by this OpenAPI file. See [Aggregator integration](/docs/aggregator-integration/overview).

Use the OpenAPI file for the v1 endpoint paths, headers, schemas, examples, and error shapes. Use the integration docs for the live contract and the operational rules that OpenAPI cannot express:

- settlement-once behavior
- idempotency retention and atomic ledger handling
- timeout-after-apply retries
- responsible-gaming and account-blocking behavior
- launch-token handling and iframe security

## Security schemes

The v1 spec models the launch bearer token and callback headers:

- `Authorization: Bearer <token>`
- `X-Spark-Signature`
- `X-Spark-Timestamp`
- `X-Spark-Request-Id`

The callback signature canonical string is:

```text
{timestamp}.{METHOD}.{path-and-query}.{rawBody}
```

See [Wallet API authentication](/docs/operator-integration/wallet-api#authentication) for the live auth model.
