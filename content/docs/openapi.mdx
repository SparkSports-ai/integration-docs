Use the OpenAPI file for endpoint paths, headers, schemas, examples, and error shapes. Use the integration docs for operational rules:

- settlement-once behavior
- idempotency retention and atomic ledger handling
- timeout-after-apply retries
- responsible-gaming and account-blocking behavior
- launch-token handling and iframe security

{/* v1: explicit boundary between OpenAPI and prose docs. WHY: OpenAPI is good for request/response validation but weak at expressing temporal money rules like "retry must return applied result after timeout" and "a closed round cannot move money again." */}

## Security schemes

The OpenAPI spec models the launch bearer token and callback headers:

- `Authorization: Bearer <token>`
- `X-Spark-Signature`
- `X-Spark-Timestamp`
- `X-Spark-Request-Id`

The callback signature canonical string is:

```text
{timestamp}.{METHOD}.{path-and-query}.{rawBody}
```

See [Wallet API authentication](/docs/direct-integration/wallet-api#authentication) for verification details.
