Build and test against staging first. Production credentials are separate and we usually hand them over after staging sign-off.

## Launch API

| Environment | Base URL |
| --- | --- |
| Production | `https://iframe-launcher.onrender.com` |
| Staging | `https://iframe-launcher-staging.onrender.com` |

## Game frontend (iframe)

| Environment | URL |
| --- | --- |
| Staging | `https://staging.spark.sparksports.ai` |
| Production | Sent during onboarding |

## What we send per environment

| Item | Description |
| --- | --- |
| Launch credentials | Bearer token for `POST /api/game/launch` |
| Callback credentials | Bearer token + signing secret for our calls to your wallet API |
| Operator slug | Your operator identifier |
| Stake limits | Min/max stake and preset amounts per currency |
| Win caps | Max multiplier and max win amount |

Email your SparkSports contact to get staging access.
