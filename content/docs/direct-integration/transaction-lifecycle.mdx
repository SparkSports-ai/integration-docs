---
title: Transaction lifecycle
description: How bets, wins, and rollbacks flow through a Spark game round.
---

Each round gets a `RoundId`. Most rounds have one bet and one win or rollback. Some paths add a zero-amount win on loss.

## Flow

```
Player starts streak
        │
        ▼
   Type 1 (Bet)          debit stake
        │
        ▼
   Player picks markets
        │
        ├── Player loses ──► Type 2 (Win, amount=0) ──► round done
        ├── Player cashes out ──► Type 2 (Win) ──► credit winnings
        └── Error / void ──► Type 3 (Rollback) ──► refund bet
```

## Transaction types

| Type | Action | When |
| --- | --- | --- |
| `1` | Debit | New streak starts |
| `2` | Credit | Cashout or round ends |
| `3` | Refund | Bet must be reversed |

## Typical sequence

1. Player opens Spark. Your backend calls [Launch](/docs/direct-integration/launch).
2. We hit [Session validation](/docs/direct-integration/session-validation).
3. Player starts a streak. Type 1 debits the stake.
4. Player picks through live steps.
5. Cashout or loss: Type 2 credits winnings (`0` on loss).
6. Void or error: Type 3 refunds the original bet.

## Retries

We retry wallet calls when the failure looks transient. Your handler must be idempotent on `ExternalTransactionId` so a retry never double-charges or double-credits.
