---
title: Launch the Game
description: Request a signed iframe URL when a player opens Spark.
---

When a logged-in player opens Spark, your backend calls our launch endpoint and gets back a URL to load in an iframe.

## Endpoint

Production:

```
POST https://iframe-launcher.onrender.com/api/game/launch
```

Staging:

```
POST https://iframe-launcher-staging.onrender.com/api/game/launch
```

## Authentication

HTTP Basic Auth. We give you the username and password during onboarding.

## Request

```json
{
  "SessionId": "your-player-session-id"
}
```

`SessionId` is whatever session token your platform already uses for that player. We pass it to your authenticate endpoint to confirm the session before issuing a URL.

## Response

```json
{
  "url": "https://game.sparksports.ai/game/sparksports?jwt=eyJhbGci...",
  "expiresIn": 30
}
```

| Field | Description |
| --- | --- |
| `url` | Iframe URL for Spark, with a short-lived token in the query string |
| `expiresIn` | Token lifetime in seconds (default 30) |

Treat the token as single-use. Request a new URL each time someone starts a session.

## Embed the game

Put the URL in an iframe:

```html
<iframe
  src="https://game.sparksports.ai/game/sparksports?jwt=..."
  width="100%"
  height="700"
  frameborder="0"
  allow="fullscreen"
></iframe>
```

1200 × 800 px works well as a max size. The UI scales down from there.

## Errors

| HTTP | Meaning |
| --- | --- |
| `401` | Bad launch credentials |
| `400` | Missing `SessionId` |
| `502` | Session validation failed ([Session validation](/docs/direct-integration/session-validation)) |
| `503` | Launch disabled (maintenance) |
