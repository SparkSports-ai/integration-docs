---
title: Launch the game
description: Request a signed iframe URL when a player opens Spark.
---

When a logged-in player opens Spark, your backend calls our launch endpoint and gets back a URL to load in an iframe.

<Callout type="info">**Proposed for v1.** Bearer auth and the absolute `expiresAt` field below are the target v1 contract and need backend work before they go live.</Callout>

## Endpoint

Production:

```
POST https://iframe-launcher.onrender.com/api/game/launch
```

Staging:

```
POST https://iframe-launcher-staging.onrender.com/api/game/launch
```

## Authentication

**Bearer token** (`Authorization: Bearer <token>`). We issue it during onboarding. Call this endpoint only from your backend; never from browser code.

## Request

```json
{
  "sessionId": "your-player-session-id"
}
```

`sessionId` is whatever session token your platform already uses for that player. We pass it to your authenticate endpoint to confirm the session before issuing a URL.

## Response

```json
{
  "url": "https://staging.spark.sparksports.ai/game/sparksports?jwt=eyJhbGci...",
  "expiresAt": 1782763200
}
```

| Field | Description |
| --- | --- |
| `url` | Iframe URL for Spark, with a short-lived token in the query string |
| `expiresAt` | Token expiry as a UNIX timestamp (seconds). Same value as the JWT `exp` claim |

The token is short-lived (about 30 seconds). Request a new URL each time a player opens Spark; do not cache or share it.

Log the launch request without the JWT query string. Treat the returned `url` as sensitive until it expires.

## Embed the game

Put the URL in an iframe:

```html
<iframe
  src="https://staging.spark.sparksports.ai/game/sparksports?jwt=..."
  width="100%"
  height="700"
  frameborder="0"
  allow="fullscreen"
></iframe>
```

1200 x 800 px works well as a max size. The UI scales down from there.

Recommended iframe attributes:

- Use `allow="fullscreen"` only; do not grant camera, microphone, geolocation, clipboard, or payment permissions.
- Set the parent page `referrer-policy` to `strict-origin-when-cross-origin` or stricter.
- Use `frame-src` / `child-src` in your Content Security Policy to allow only the Spark iframe origins we give you.
- Do not put the launch URL in analytics events, frontend logs, crash reports, or support screenshots.

## Errors

| HTTP | `error.code` | Meaning |
| --- | --- | --- |
| `400` | `MISSING_SESSION_ID` | Request had no `sessionId` |
| `401` | `UNAUTHORIZED` | Bad bearer token |
| `403` | `OPERATOR_INACTIVE` | Operator account inactive |
| `502` | `SESSION_VALIDATION_FAILED` | Your [authenticate](/docs/direct-integration/session-validation) call failed; we pass through the reason |
| `503` | `LAUNCH_DISABLED` | Launch disabled (maintenance) |
| `429` | `RATE_LIMITED` | Too many launch attempts for the operator or source |
