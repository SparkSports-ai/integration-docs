---
title: Wallet API
description: Endpoints your platform must expose for bets, wins, and rollbacks.
---

While someone is playing Spark, we call your wallet API to move money. Every call is synchronous: we send the request, wait for your response, then continue or show an error to the player.

## Authentication

HTTP Basic Auth on all wallet endpoints, using the callback credentials from onboarding.

## Get balance

We call this when we need a fresh balance, usually after a failed transaction retry.

```
GET {your-callback-base-url}/api/generic/user/{pincode}/balance?sessionId={sessionId}
```

Response:

```json
{
  "Balance": 1500.00
}
```

## Process transaction

One endpoint for bets, wins, and rollbacks.

```
POST {your-callback-base-url}/api/transaction/process
```

Request body:

```json
{
  "Amount": 10.00,
  "Type": 1,
  "ExternalTransactionId": "spark-tx-abc123",
  "Description": "Spark streak bet",
  "GameId": "sparksports",
  "RoundId": "round-uuid-here",
  "PinCode": "player-unique-id",
  "SessionId": "your-player-session-id"
}
```

| Field | Description |
| --- | --- |
| `Amount` | Always positive |
| `Type` | `1` bet (debit), `2` win (credit), `3` rollback (refund) |
| `ExternalTransactionId` | Our transaction ID; use this for idempotency |
| `Description` | Free text |
| `GameId` | Always `"sparksports"` |
| `RoundId` | UUID for the round |
| `PinCode` | From authenticate |
| `SessionId` | Active player session |

Response:

```json
{
  "TransactionId": 987654321,
  "CurrentBalance": 1490.00,
  "ExternalTransactionId": "spark-tx-abc123"
}
```

## Idempotency

Same `ExternalTransactionId` twice should return the same result without processing twice. We retry on network blips.

## Errors

```json
{
  "ErrorCode": 201,
  "Message": "Insufficient balance"
}
```

| ErrorCode | HTTP | Meaning |
| --- | --- | --- |
| `102` | 403 | Session expired |
| `111` | 403 | Player blocked |
| `201` | 400 | Not enough balance |
| `202` | 400 | Duplicate transaction |

We retry transient failures (network errors, 5xx) up to three times with backoff. We do not retry insufficient balance or expired session; those go straight to the player.
