## Requirements

| Requirement | Detail |
| --- | --- |
| TLS | HTTPS on every API call |
| Credential storage | Treat launch and callback passwords like production secrets; rotate if leaked |
| Session isolation | Do not expose `SessionId` to the game client |
| Idempotency | Key wallet handling on `ExternalTransactionId` |
| Launch usage | Request one launch URL per player session. Do not poll the endpoint |

## Launch tokens

Launch URLs carry a JWT that expires in about 30 seconds by default.

- Request a new URL per session; do not reuse tokens
- Balance is not in the JWT; we pull it from your wallet API during play

## IP allowlisting (optional)

If you want to lock down wallet callbacks to our egress IPs, ask during onboarding. We can send the addresses to allowlist.

## This documentation site

Run it behind an access gate (Cloudflare Zero Trust works well). Do not leave it public. Credentials always go out-of-band, never in these pages.
