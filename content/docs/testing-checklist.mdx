This checklist is for the [operator integration](/docs/operator-integration/launch) path. If you integrate through an aggregator, you do not build these endpoints; see [Aggregator integration](/docs/aggregator-integration/overview).

Run through this in staging before asking for production keys. Replace the `$LAUNCH_*`, `$CALLBACK_*`, `$PINCODE`, and `$SESSION_ID` placeholders with your staging values.

## Launch

**Valid session returns a game URL**

Call launch with a real session. Expect a 200 with `url` in the body.

```bash
curl -X POST https://iframe-launcher-staging.onrender.com/api/game/launch \
  -u "$LAUNCH_USER:$LAUNCH_PASS" \
  -H "Content-Type: application/json" \
  -d '{"SessionId":"<valid-session>"}'
```

- [ ] Valid session returns `url`
- [ ] Bad Basic Auth returns `401 Unauthorized`
- [ ] Missing `SessionId` returns `400 BadRequest`
- [ ] Launch endpoint is not callable from browser code with exposed credentials
- [ ] Full launch URL and JWT are absent from frontend logs, analytics, and crash reports
- [ ] Game loads in iframe on desktop
- [ ] Game loads in iframe on mobile

## Session validation

Replay these against your own handler to confirm it returns the right shape.

**Valid session returns player fields from authenticate**

Expect a 200 with `SessionId`, `Pincode`, `Username`, `Balance`, `Currency`, and optionally `Country`.

```bash
curl -X POST "$CALLBACK_BASE/api/generic/authenticate" \
  -u "$CALLBACK_USER:$CALLBACK_PASS" \
  -H "Content-Type: application/json" \
  -d '{"SessionId":"<valid-session>"}'
```

- [ ] Valid session returns `SessionId`, `Pincode`, `Username`, `Balance`, `Currency`
- [ ] Expired session returns `ErrorCode: 102`
- [ ] Blocked player returns `ErrorCode: 111`
- [ ] `Country` is optional; `Balance`, `Username`, and `Currency` are required
- [ ] Frozen, self-excluded, and jurisdiction-blocked accounts cannot create playable sessions

## Wallet

Replay these against your own handler. Use a fresh `ExternalTransactionId` for each call.

**Type 1 debits the right amount**

Send a bet. Expect a 200 with `TransactionId`, `CurrentBalance` (reduced by the stake), and the echoed `ExternalTransactionId`.

```bash
curl -X POST "$CALLBACK_BASE/api/transaction/process" \
  -u "$CALLBACK_USER:$CALLBACK_PASS" -H "Content-Type: application/json" \
  -d '{"Amount":10.00,"Type":1,"ExternalTransactionId":"tx-1","Description":"bet","GameId":"sparksports","RoundId":"<uuid>","PinCode":"<pin>","SessionId":"<session>"}'
```

**Type 2 with amount `0` on loss**

Send a zero-amount win. Expect a 200 and `CurrentBalance` unchanged.

```bash
curl -X POST "$CALLBACK_BASE/api/transaction/process" \
  -u "$CALLBACK_USER:$CALLBACK_PASS" -H "Content-Type: application/json" \
  -d '{"Amount":0,"Type":2,"ExternalTransactionId":"tx-3","Description":"loss","GameId":"sparksports","RoundId":"<uuid>","PinCode":"<pin>","SessionId":"<session>"}'
```

**Type 3 refunds the original bet**

Send a rollback. Expect `CurrentBalance` restored to the pre-bet value.

```bash
curl -X POST "$CALLBACK_BASE/api/transaction/process" \
  -u "$CALLBACK_USER:$CALLBACK_PASS" -H "Content-Type: application/json" \
  -d '{"Amount":10.00,"Type":3,"ExternalTransactionId":"tx-4","Description":"rollback","GameId":"sparksports","RoundId":"<uuid>","PinCode":"<pin>","SessionId":"<session>"}'
```

- [ ] Type 1 debits the right amount
- [ ] Type 2 credits the right amount
- [ ] Type 2 with amount `0` on loss closes the round without crediting
- [ ] Type 3 refunds the full original stake, never a partial amount
- [ ] Duplicate `ExternalTransactionId` is idempotent (same `TransactionId` and `CurrentBalance`, no second move)
- [ ] Insufficient balance returns `ErrorCode: 104`
- [ ] Balance endpoint returns `Balance` matching wallet state
- [ ] Concurrent calls for the same player are handled safely (balance read during a bet, bets for two streaks)

## End-to-end

These run through the live game, not curl.

- [ ] Launch, bet, cashout, win reflected in balance
- [ ] Launch, bet, loss, round closed (Type 2 amount `0`)
- [ ] Bet rolled back on void (Type 3)
- [ ] All wallet logs include `ExternalTransactionId`, `RoundId`, `PinCode`, and `SessionId`

When staging looks good, ping your SparkSports contact for production provisioning.
